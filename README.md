## sanity checks (შემოწმება ტრენინგის დაწყებამდე)

უშუალოდ მოდელის ტრენინგის დაწყებამდე ვატარებ სამ ძირითად ტესტს, რათა დავრწმუნდე კოდის გამართულობასა და სისწორეში:
- Forward Pass: ვამოწმებთ, არის თუ არა გამომავალი output-ის ზომა სწორი — მოცემულ შემთხვევაში (8, 7). ასევე მოწმდება initial loss, რომელიც დაახლოებით $ln(7) \approx 1.946$-ის ტოლი უნდა იყოს. ეს ტესტი ამოწმებს, რომ წონების ინიციალიზაცია სწორად მოხდა და, შესაბამისად, საწყის ეტაპზე მოდელი (შემთხვევითი წონების გამო) ყველა კლასს თანაბარი ალბათობით წინასწარმეტყველებს.
- Backward: ვამოწმებთ, რომ მოდელის ყველა პარამეტრი იღებს არანულოვან გრადიენტს (non-zero gradient). ეს კრიტიკულად მნიშვნელოვანია, რადგან გვეხმარება ისეთი ფარული შეცდომების დროულ აღმოჩენაში, როგორიცაა "მკვდარი ნეირონები" (dead neurons) და გაწყვეტილი გამოთვლითი გრაფი (broken computation graph).
- Overfit on 1 Batch: ვტესტავთ, შეუძლია თუ არა TinyNet მოდელს 100 ბიჯის (gradient step) შემდეგ მცირე მონაცემებზე — 1 ბეტჩზე (32 სურათზე) — სიზუსტის თითქმის 100%-მდე აყვანა. ამ დროს მონაცემთა აუგმენტაცია გათიშულია (augment=False). ეს არის საუკეთესო გზა იმის დასადასტურებლად, რომ ლოს ფუნქცია და training loo ტექნიკურად სწორია. ა
-

## Data Augmentation
ტრენინგის პროცესში ვიყენებ შემდეგ ტრანსფორმაციებს, რათა მოდელმა უკეთ შეძლოს განზოგადება და თავიდან ავიცილოთ overfitting:

- *RandomHorizontalFlip (p=0.5):* ამ ტექნიკით ხდება ფოტოების სარკისებურად შებრუნება, რის შედეგადაც ხელოვნულად იზრდება მონაცემების რაოდენობა. ჰორიზონტალურად შებრუნებულ ფოტოებში იგივე ემოციაა გამოსახული, რაც შებრუნებამდე იქნებოდა. ეს გვეხმარება, რომ მოდელმა ტრენინგისას არ დაიზეპიროს მონაცემები, ან სხვა დეტალებზე, რომლებიც ემოციასთან არ იქნება კავშირში, არ გაამახვილოს ყურადღება შეცდომით. 
- *RandomRotation (±10°):* რადგან ფოტოებში ადამიანები სხვადასხვა კუთხით ჩანან და არა ზუსტად სწორად, მცირე როტაციის დამატება მოდელს ეხმარება, რომ ემოცია ოდნავ დახრილ სახეებზეც ზუსტად ამოიცნოს და არ დაიბნეს.
- *RandomCrop (size=48, padding=4):* 48x48 ზომის სურათს ვამატებთ 4 padding-ს, რომელიც ხელოვნურად ზრდის სურათის ზომას, შეწმდეგ ამ გადიდებულ ფოტოს random ადგილზე ვჭრით და ვიღებთ ისევ 48x48 ფოტოს. ამის გამო ვიღებთ ფოტოს ოდნავ გადანაცვლებული კოორდინატებით, თუმცა მნიშვნელოვან დეტალებს სახეზე არ ვკარგავთ. ეს ცვლილება იმისთვისაა მნიშვნელოვანი, რომ მოდელმა არ დაიზეპიროს კონკრეტული კოორდინატები სახის და ამის მიხედვით არ გამოიტანოს დასკვნა. 
- *ColorJitter (brightness=0.2, contrast=0.2):* შემთხვევითად ცვლის ფოტოების brightness და contrast-ს, პიქსელების რენდომ რიცხვზე გამრავლებით (+-20%), რომ მოდელმა ნებისმიერ განათებაზე შეძლოს ემოციის ამოცნობა. 

Validation და Test სეტები
სატესტო და ვალიდაციის მონაცემებზე ზემოთ ჩამოთვლილი ცვლილებები არ ვრცელდება. მათზე ხდება მხოლოდ საბაზისო ტრანსფორმაციები (ტენზორად გადაქცევა და ნორმალიზაცია), რათა მოდელის მუშაობის შეფასება მოხდეს რეალურ, დაუმახინჯებელ მონაცემებზე.

## Learning Rate Schedulers

- CosineAnnealingLR: ეს მეთოდი სწავლების ტემპს თანმიმდევრულად, კოსინუსის მრუდის მიხედვით ამცირებს თითქმის ნულამდე. საწყის ეტაპზე მაღალი ტემპი მოდელს ეხმარება სწრაფად ისწავლოს ზოგადი მახასიათებლები, ხოლო ბოლო ეპოქებში პატარა ნაბიჯები საშუალებას აძლევს, ზედმეტი რხევების გარეშე ზუსტად მიაგნოს ოპტიმუმს და მაქსიმალურად გააუმჯობესოს საბოლოო კონვერგენცია.

- ReduceLROnPlateau: ეს არის ადაპტირებადი მიდგომა. ალგორითმი აკვირდება სავალიდაციო დანაკარგს. თუ მოდელის პროგრესი ზედიზედ 5 ეპოქის განმავლობაში აღარ მიმდინარეობს(patience=5), ის სწავლების ტემპს ანახევრებს (factor=0.5). ეს განსაკუთრებით ეფექტურია მე-8 ექსპერიმენტში (Transfer Learning), სადაც ბაზისური ფენები გაყინულია (frozen head) და მხოლოდ ბოლო ფენების სწრაფი, მაგრამ სტაბილური ადაპტაცია გვჭირდება.

- None (მუდმივი LR): 1-ელი და მე-2 ექსპერიმენტები წარმოადგენს ე.წ. clean baseline-ს, სადაც ვაკვირდები თავად არქიტექტურის ქცევას დამატებითი ოპტიმიზაციების გარეშე. მე-5 ექსპერიმენტში კი ვიყენებ მუდმივ ტემპს, რათა სხვადასხვა ოპტიმიზატორის მუშაობა ობიექტურად შევადარო და შედეგებზე Scheduler-მა არ მოახდინოს გავლენა.

## არქიტექტურების პროგრესი

მიდგომა სრულიად იტერაციულია — თითოეული ექსპერიმენტი კონკრეტულ პრობლემას ავლენს, შემდეგი ექსპერიმენტი კი ამ პრობლემას ასწორებს:

```
Exp 1:  TinyNet            → underfitting       → მეტი layer-ი სჭირდება
Exp 2:  SmallCNN (d=0)     → overfitting        → რეგულარიზაცია სჭირდება
Exp 3:  SmallCNN (d=0.4)   → გაუმჯობესებული    → სიღრმე გასაზრდელია
Exp 4:  MediumCNN + Adam   → დაბალანსებული     → optimizer-ის შედარება
Exp 5:  MediumCNN + SGD    → Adam იმარჯვებს     → LR-ის ტესტირება
Exp 6:  MediumCNN LR↑      → არასტაბილური      → residual connections
Exp 7:  DeepCNN            → საუკეთესო CNN      → transfer learning
Exp 8:  ResNet frozen      → შეზღუდული        → სრული fine-tune
Exp 9:  ResNet finetune    → CNN-ზე ნაკლები     ->არ გააუმჯობესა საუკეთესო მოდელის შედეგი
```

## ექსპერიმენტები

### ექსპერიმენტი 1 — TinyNet საბაზისო მოდელი

**არქიტექტურა:**
2 conv layer-ი 8 და 16 ფილტრით
```
Input(1×48×48)
  → Conv2d(1→8, 3×3, pad=1) → ReLU → MaxPool2d(2×2)   # 8×24×24
  → Conv2d(8→16, 3×3, pad=1) → ReLU → MaxPool2d(2×2)  # 16×12×12
  → Flatten → FC(2304→7)
```

**პარამეტრების რაოდენობა:** 37,255

- optimizer=Adam, lr=0.001 — ადამის სტანდარტული მნიშვნელობა.
- batch=64, epochs=30 — მცირე ეპოქების რაოდენობა საკმარისია; მეტი ეპოქა underfitting პრობლემას ვერ მოაგვარებს.
- augmentation=False, dropout=0, weight_decay=0 — რეგულარიზაციას არ ვიყენებთ.
  
**შედეგი:**
```
Val: 50.63%
Test: 49.60%
```
**ანალიზი:** მოდელი ავლენს კლასიკურ Underfitting-ს. Train და Val სიზუსტეები ერთმანეთთან ახლოსაა და ორივე დაბალია. მოდელი ძალიან მარტივია (მოკლებულია სიმძლავრეს), ამიტომ საჭიროა ფენებისა და ფილტრების რაოდენობის გაზრდა.

### ექსპერიმენტი 2 — SmallCNN (Dropout-ის გარეშე)

**არქიტექტურა:**
3 Conv ბლოკი BatchNorm-ით + Flatten + 2 FC ფენა.

```
Input(1×48×48)
  → Conv2d(1→32, 3×3, pad=1) + BN + ReLU + MaxPool2d(2×2)   # 32×24×24
  → Conv2d(32→64, 3×3, pad=1) + BN + ReLU + MaxPool2d(2×2)  # 64×12×12
  → Conv2d(64→128, 3×3, pad=1) + BN + ReLU + MaxPool2d(2×2) # 128×6×6
  → Flatten
  → Dropout(0.0) → FC(128×6×6=4608 → 512) → ReLU
  → Dropout(0.0) → FC(512 → 7)
```

Dropout(0.0) ნიშნავს, რომ Dropout layer-ები კოდში არსებობს, მაგრამ p=0-ით გამორთულია. რადგან ადვილად შევადაროთ შემდეგ მოდელებს და დავინახოთ დროფაუთის გავლენა.
 
**პარამეტრების რაოდენობა:** 1,234,247

**კონფიგურაცია და დასაბუთება:**
- `optimizer=Adam, lr=0.001, epochs=50` 
- `augmentation=False, dropout=0, weight_decay=0` — regularization გამორთული.
-  `batch_size=64, weight_decay=0` — L2 regularization-ის გარეშე

**შედეგი:**
```
Train: 98.81% 
Val : 60.24%
Test : 60.32%
```

**ანალიზი:** ამ ექსპერიმენტში იყო 33-ჯერ მეტი პარამეტრი, ვიდრე პირველში, რაც წესით მოდელის სიზუსტისთვის კარგია, მაგრამ WandB გრაფიკებზე ჩანს Overfitting-ის მაგალითი: Train სიზუსტე ბოლო ეპოქებში თითქმის 100% ხდება, Val კი 60%-ზე ჩერდება და დაახლოებით 25-ე ეპოქიდან მისი დანაკარგი იზრდება. საბოლოო ჯამში, არქიტექტურა მუშაობს (+10% გაუმჯობესება პირველთან შედარებით), მაგრამ იზეპირებს სატრენინგო მონაცემებს და სჭირდება რეგულარიზაცია.

### ექსპერიმენტი 3 — SmallCNN (Dropout + Augmentation)

**არქიტექტურა:** Exp 2-ის იდენტური

**კონფიგურაცია:**
- dropout=0.4 (FC ფენების წინ): წარმოადგენს ოპტიმალურ ბალანსს (trade-off) რეგულარიზაციასა და მოდელის სწავლის უნარს შორის. ზედმეტად მაღალი dropout (>0.5) გამოიწვევდა ინფორმაციის დაკარგვასა და Underfitting-ს.
- augmentation=True: ტრანსფორმაციები (RandomFlip, Rotation, Crop, ColorJitter) ხელოვნურად ზრდის სატრენინგო მონაცემთა ეფექტურ მოცულობასა და მრავალფეროვნებას, რაც მოდელს უცხო მონაცემებზე განზოგადებაში ეხმარება.
- weight_decay=1e-4: მსუბუქი L2 რეგულარიზაცია, მოდელს აჯარიმებს ზედმეტად დიდი წონების შემთხვევაში. dropout და ეს ერთად მოდელს აიძულებს ისწავლოს მხოლოდ ყველაზე მნიშვნელოვანი და განზოგადებადი მახასიათებლები.
- scheduler=CosineAnnealing: სწავლების ტემპს ნელა ამცირებს, რაც უზრუნველყოფს ლოკალური მინიმუმისკენ სტაბილურ კონვერგენციას.

ჰიპოთეზა: 0.4-იანი Dropout დაშლის ნეირონების არასასურველ co-adaptation-ს, ხოლო მონაცემთა აუგმენტაცია გაზრდის უნიკალური ნიმუშების რაოდენობას. ამ ორი ძლიერი რეგულარიზაციის მეთოდის კომბინაცია შეამცირებს სხვაობას Train და Val სიზუსტეებს შორის, რაც წინა ექსპერიმენტში (Exp 2) მთავარ პრობლემას წარმოადგენდა.

**რეალური შედეგი:**
```
Train: 63.69%
Val : 64.17%
Test : 65.37%
```

**ანალიზი:** ჰიპოთეზა გამართლდა — აუგმენტაციამ და Dropout-მა მოდელი დაარეგულირა. მე-2 ექსპერიმენტთან შედარებით, სადაც overfitting ფიქსირდებოდა, Train და Val მრუდებს შორის სხვაობა მკვეთრად შემცირდა. ამასთან, სავალიდაციო სიზუსტე 60.24%-დან 64.17%-მდე გაიზარდა. მიუხედავად იმისა, რომ რეგულარიზაციამ მოდელი ბევრად სტაბილური გახადა, მიღწეული ჭერი მიანიშნებს, რომ 3 კონვოლუციური ბლოკი მაინც ზღუდავს მოდელის პოტენციალს. ამიტომ, სიზუსტის შემდგომი ზრდისთვის ლოგიკური ნაბიჯი არქიტექტურის სიღრმის გაზრდაა.

### ექსპერიმენტი 4 — MediumCNN + Adam

**არქიტექტურა:**
```
Input(1×48×48)
  Block 1: Conv(1→32)+BN+ReLU + Conv(32→32)+BN+ReLU + MaxPool + Dropout2d(0.1)   # 32×24×24
  Block 2: Conv(32→64)+BN+ReLU + Conv(64→64)+BN+ReLU + MaxPool + Dropout2d(0.1)  # 64×12×12
  Block 3: Conv(64→128)+BN+ReLU + Conv(128→128)+BN+ReLU + MaxPool + Dropout2d(0.2) # 128×6×6
  Block 4: Conv(128→256)+BN+ReLU + MaxPool                                         # 256×3×3
  → Flatten → Dropout(0.4) → FC(2304→512) → ReLU → Dropout(0.4) → FC(512→256) → ReLU → FC(256→7)
```

კონვოლუციური ბლოკები: 4-ეტაპიანი არქიტექტურა VGG-ს სტილის ორმაგი კონვოლუციური ბლოკებით (პირველი 3 ბლოკი) და ერთი საბოლოო კონვოლუციური ბლოკით.
რეგულარიზაცია: გამოყენებულია Spatial Dropout2d (feature map-ების გათიშვა) თითოეულ ბლოკში და Dropout(0.4) სრულად დაკავშირებულ (FC) ფენებში.
**პარამეტრების რაოდენობა:** 2,847,879

**კონფიგურაცია:**
- optimizer=Adam, lr=0.001: ადაპტური სწავლების ტემპი ეხმარება მოდელს გრადიენტების მრავალფეროვან მასშტაბთან ადაპტაციაში.
- weight_decay=1e-4: მსუბუქი L2 რეგულარიზაცია, რომელიც Dropout-თან ერთად აფერხებს წონების გადამეტებულ ზრდას.
- epochs=60: მოდელის გაზრდილი სიღრმის გამო, საჭიროა დამატებითი ეპოქები კონვერგენციისთვის.
- scheduler=CosineAnnealing: გლუვი შემცირება LR-ის ნულამდე, რაც აუმჯობესებს მოდელის სტაბილურობას სწავლების ბოლო ეტაპზე.
- batch=64: ოპტიმალური ბალანსი მეხსიერების გამოყენებასა და გრადიენტის სტაბილურობას შორის.

**ჰიპოთეზა:** ორმაგი კონვოლუციური ბლოკების დამატება საშუალებას მისცემს ქსელს, ამოიღოს უფრო კომპლექსური ნიშნები. Spatial Dropout-ის გამოყენება თავიდან აგვაცილებს redundant მახასიათებლების დაზეპირებას, რაც გააუმჯობესებს განზოგადების უნარს.

**რეალური შედეგი:**
```
Train:  70.31%
Val : 66.51%
Test : 69.41%
```

**ანალიზი:** 
- MediumCNN უკეთესი აღმოჩნდა SmallCNN-თან შედარებით (+2.34% საუკეთესო Val-ზე). მოდელმა შეძლო მონაცემების უფრო ღრმა სტრუქტურების ათვისება.
- Train vs Val: Train და Val სიზუსტეები ერთმანეთთან ახლოსაა, რაც ადასტურებს, რომ რეგულარიზაციის სტრატეგია (Spatial Dropout + L2) ადეკვატურია და ეფექტურად უპირისპირდება Overfitting-ს.
- Spatial Dropout-ის როლი: ვინაიდან Test სიზუსტე (69.41%) აღემატება Val სიზუსტეს (66.51%), ეს მიუთითებს მოდელის ძლიერ განზოგადების უნარზე — მან ისწავლა რეალური, ფუნდამენტური ნიშნები და არა უბრალოდ სავალიდაციო სეტის თავისებურებები.
- გადაწყვეტილება: სიღრმე დადებითად აისახება შედეგზე. ეს მოდელი არის მყარი ბეისლაინი. ამის შემდეგ შევადარებ ოპტიმიზატორებს (Adam vs SGD), რათა დავრწმუნდე, რომ ჩვენი შედეგი მხოლოდ არქიტექტურის გამოა და არა კონკრეტული ოპტიმაიზერის.

### ექსპერიმენტი 5 — MediumCNN + SGD (Optimizer-ის შედარება)

**არქიტექტურა:**  მე-4 ექსპერიმენტის იდენტური

**კონფიგურაცია:**
- optimizer=SGD+Nesterov (momentum=0.9): ნესტეროვის იმპულსი (Nesterov Accelerated Gradient) ითვლის "წინასწარ გადადგმულ ნაბიჯს" გრადიენტის მიმართულებით, რაც აჩქარებს კონვერგენციას და ამცირებს ოსცილაციებს.
- lr=0.01: Adam-თან შედარებით 10-ჯერ მაღალი LR, რადგან SGD-ს არ გააჩნია ადაპტური სწავლების ტემპი თითოეული პარამეტრისთვის და საჭიროებს უფრო "აგრესიულ" გლობალურ ნაბიჯს.
- მიზანია, რომ ოპტიმიზატორების გავლენა შევადაროთ ერთმანეთს.


**ჰიპოთეზა:** მიუხედავად იმისა, რომ ადაპტური მეთოდები (Adam) სწრაფად აღწევენ "პლატოს", SGD+Nesterov-ს ხშირად უკეთესი განზოგადების უნარი აქვს, რადგან ის უფრო "ფრთხილად" მოქმედებს.

**რეალური შედეგი:**
```
Train: 63.85% 
Val : 63.39%
Test : 66.09%
```

**ანალიზი:** 
- Adam (66.51%) SGD-ს (63.39%) 3.12%-ით სჯობს.
- SGD-ს შემთხვევაში 60 ეპოქა არ აღმოჩნდა საკმარისი ოპტიმალური წონების მოსაძებნად. ის სავარაუდოდ უფრო მეტ ეპოქას ან Learning Rate Warmup-ს საჭიროებდა.
- მოცემული დატასეტი (FER2013) შედარებით მცირე და ხმაურიანია (noisy). Adam-ის ადაპტური სწავლების ტემპი ეხმარება მოდელს, სწრაფად მოერგოს განსხვავებული სიდიდის გრადიენტებს, რაც მცირე მონაცემთა ბაზებზე უპირატესია.

გადაწყვეტილება:
ექსპერიმენტმა დაადასტურა, რომ ამ არქიტექტურისა და მონაცემების პირობებში Adam/AdamW არის ბევრად უფრო ეფექტური არჩევანი. შემდგომი ექსპერიმენტებისთვის Adam/AdamW ოპტიმიზატორს გამოვიყენებ სტანდარტულად, რომ გავაუმჯობესო არქიტექტურული ოპტიმიზაციები (მაგ. Residual ბლოკები).

### ექსპერიმენტი 6 — MediumCNN მაღალი LR-ით (LR Ablation)

**არქიტექტურა:** იდენტური Exp 4-ის

**კონფიგურაცია და დასაბუთება:**
ყველა პარამეტრი იგივეა, რაც მე-4-ში, გარდა სწავლების ტემპისა, რომელიც 10-ჯერ გაიზარდა. მიზანია, რომ შევაფასოთ სწავლების ტემპის მგრძნობელობა (Sensitivity Analysis). 

ჰიპოთეზა:$lr=0.01$ მნიშვნელოვნად აღემატება Adam ოპტიმიზატორისთვის რეკომენდებულ დიაპაზონს ($10^{-4}$ - $3 \times 10^{-3}$). სავარაუდოდ, რომ მაღალი ტემპი გამოიწვევს "გადახტომას" (overshooting) მინიმუმის წერტილიდან, რაც ხელს შეუშლის კონვერგენციას.

**რეალური შედეგი:**
```
Train: 49.84% 
Val : 52.66%
Test: 53.44%
```

**ანალიზი:**
- კრიტიკული ვარდნა: მხოლოდ ერთი ჰიპერპარამეტრის ცვლილებამ (lr: 0.001 → 0.01) გამოიწვია 14%-იანი სიზუსტის ვარდნა (66.51% → 52.66%). ეს ადასტურებს, რომ სწავლების ტემპი არის ყველაზე კრიტიკული და მნიშვნელოვანი დეტალი მოდელის წარმატებაში.
- ლოსის (Loss) გრაფიკი ადასტურებს ჰიპოთეზას: Exp 4-ის გლუვ მრუდთან შედარებით, აქ ფიქსირდება მაღალი ამპლიტუდის ოსცილაციები. ოპტიმიზატორი "ხტუნავს" ლოსის ზედაპირზე და ვერ ახერხებს სტაბილურ მინიმუმში "დაჯდომას".
- რადგან Adam-ს აქვს ადაპტური სწავლების ტემპი თითოეული პარამეტრისთვის, ძალიან მაღალი გლობალური LR ამ ადაპტაციურ მექანიზმსაც კი ხელს უშლის, რაც იწვევს არასტაბილურობას მთელი ქსელის მასშტაბით.

გადაწყვეტილება: ეს ექსპერიმენტი დაგვეხმარება მოდელისთვის LR-ს სწორად არჩევაში. ჰიპერპარამეტრების ოპტიმიზაცია უნდა დაიწყოს LR-ის სწორი განსაზღვრით. Adam-ისთვის რეკომენდებული დიაპაზონია $[10^{-4}, 3 \times 10^{-3}]$. ამ მნიშვნელობების გაზრდა წინა ექსპერიმენტებში მიღწეულ პროგრესს აქრობს.

### ექსპერიმენტი 7 — DeepCNN Residual Block-ებით

**არქიტექტურა:**
```
Input(1×48×48)
  Stem:   Conv2d(1→64, 3×3) + BN + ReLU                                  # 64×48×48
  Stage1: ResBlock(64) + MaxPool                                           # 64×24×24
  Stage2: Conv(64→128) + BN + ReLU + ResBlock(128) + MaxPool              # 128×12×12
  Stage3: Conv(128→256) + BN + ReLU + ResBlock(256) + MaxPool             # 256×6×6
  Stage4: Conv(256→512) + BN + ReLU + MaxPool                             # 512×3×3
  → GlobalAvgPool → Flatten                                               # 512
  → Dropout(0.5) → FC(512→256) → ReLU → Dropout(0.25) → FC(256→7)

ResBlock(c): output = ReLU(x + [Conv(c→c)+BN+ReLU+Conv(c→c)+BN])
```
- 4-ეტაპიანი (Stage) ღრმა კონვოლუციური ქსელი, სადაც მე-2, მე-3 და მე-4 ეტაპებზე ინტეგრირებულია ResBlock (Residual/Skip კავშირები).

- Global Average Pooling (GAP): სტანდარტული Flatten-ის ნაცვლად, ბოლო სივრცული განზომილების (3×3) შესაკუმშად გამოყენებულია GAP, რაც მნიშვნელოვნად ამცირებს პარამეტრების რაოდენობას FC ფენებში (MediumCNN-ის 2304 პარამეტრიდან შემცირდა 512-მდე).

**პარამეტრების რაოდენობა:** 6,573,127

**კონფიგურაცია:**
- optimizer=AdamW, weight_decay=1e-3: ტრადიციული Adam-ისგან განსხვავებით, რომელიც L2 ჯარიმას გრადიენტის გამოთვლისას ურევს, AdamW მიჯნავს (decouples) weight_decay-ს გრადიენტის განახლებისგან. ეს უზრუნველყოფს წონების ბევრად უფრო სწორ და ეფექტურ რეგულარიზაციას.
- dropout=0.5 / 0.25: პროგრესული Dropout. feature extraction ფენის დასრულების შემდეგ ვიყენებთ ძლიერ რეგულარიზაციას (0.5), ხოლო მეორე FC ფენასთან უფრო სუსტს (0.25).
- batch=32: 64-დან შემცირებული ბეტჩის ზომა იწვევს გრადიენტების უფრო ხშირ, ოდნავ ხმაურიან განახლებას, რაც მოდელს ლოკალური მინიმუმებიდან თავის დაღწევასა და უკეთეს განზოგადებაში ეხმარება.
-ეპოქები=70: ქსელის დიდი სიღრმის გამო, ოპტიმალური კონვერგენციისთვის მეტი იტერაცია გახდა საჭირო.

ჰიპოთეზა:
ქსელის გაღრმავება ჩვეულებრივ იწვევს Vanishing Gradient პრობლემას, თუმცა შემოტანილი Residual connections გრადიენტს საშუალებას აძლევს, დაუბრკოლებლად გადაეცეს ადრეულ ფენებს. ამასთან, GAP ფენა მკვეთრად შეამცირებს პარამეტრებს გადაწყვეტილების მიმღებ ბლოკში, რაც ძლიერ AdamW-სთან ერთად შეაკავებს Overfitting-ს. სავარაუდოდ, ეს მოდელი აჩვენებს საუკეთესო შედეგს.

**რეალური შედეგი:**
```
Train: 86.73% 
Val : 68.99%
Test : 69.63%
```

**ანალიზი:** DeepCNN ყველა ექსპერიმენტს შორის საუკეთესო შედეგს (68.99%) აღწევს. Skip კავშირებმა შეძლო გაცილებით ღრმა ქსელის ვარჯიში gradient-ის გაქრობის გარეშე. GAP-მა FC პარამეტრები 512-მდე შეამცირა (MediumCNN-ში 2304 იყო), overfitting-ი შემცირდა. AdamW + weight_decay=1e-3 ძლიერი რეგულარიზაცია.
DeepCNN-ს train სიზუსტე (86.73%) val-ზე (68.99%) ბევრად მეტია, თუმცა ეს კრიტიკული overfitting არ არის — სატესტო accuracy (69.63%) ვალიდაციას აჭარბებს, რაც კარგ განზოგადებაზე მიუთითებს. Train-validation ამხელა სხვაობა FER2013-ის სირთულის გამოა, ეს dataset თვითონვეა ორაზროვანი და ბუნდოვანი. Val loss სწავლების განმავლობაში სტაბილურად მცირდებოდა, რაც ამ შედეგს ნამდვილი overfitting-სგან განასხვავებს.

**გადაწყვეტილება:** Custom CNN residual კავშირებით საუკეთესოა.

### ექსპერიმენტი 8 — ResNet18 Frozen Backbone

**არქიტექტურა:**
```
ResNet18 (ImageNet pretrained)
  conv1 შეცვლილია: Conv2d(3→64, 7×7) → Conv2d(1→64, 7×7)  [grayscale ადაპტაცია]
  layer1–layer4: frozen (trainable=False)
  fc შეცვლილია: Dropout(0.5) → FC(512→256) → ReLU → Dropout(0.3) → FC(256→7)
```
**პარამეტრების რაოდენობა:** ~530,000 (მხოლოდ პირველი ფენა და head)

- საბაზო მოდელი: ImageNet-ზე პრეტრენინგებული ResNet18.
- ადაპტაცია (Grayscale): conv1 ფენა შეიცვალა Conv2d(1→64, 7×7)-ით, რათა მოდელმა 3-არხიანი (RGB) სურათების ნაცვლად, 1-არხიანი (შავ-თეთრი) FER2013-ის მონაცემები მიიღოს.
- Freezing: layer1–დან layer4-ის ჩათვლით ყველა ფენის წონები შენარჩუნებულია (trainable=False).
- კლასიფიკატორი (Custom Head): ორიგინალი FC ფენა შეიცვალა ახალი, რეგულარიზებული ბლოკით: Dropout(0.5) → FC(512→256) → ReLU → Dropout(0.3) → FC(256→7).

კონფიგურაცია:
- ეპოქები=20: რადგან ქსელის უდიდესი ნაწილი გაყინულია და მხოლოდ თავი იწვრთნება, კონვერგენციისთვის ნაკლები ეპოქაც საკმარისი უნდა იყოს.
- optimizer=Adam, lr=0.001: უზრუნველყოფს წონების სწრაფ ადაპტაციას.
- scheduler=ReduceLROnPlateau: დინამიური მიდგომა LR-ის სამართავად — თუ val. loss ზედიზედ 5 ეპოქის განმავლობაში არ გაუმჯობესდა (Plateau), სწავლების ტემპი ავტომატურად განახევრდება.

ჰიპოთეზა:
ტრანსფერული სწავლების (Transfer Learning) კლასიკური მიდგომა: მიუხედავად იმისა, რომ მოდელი სხვა ინფუთებზე(ImageNet) დატრენინგდა, შეიძლება, მის მიერ ნასწავლი საბაზისო ვიზუალური მახასიათებლები — კიდეები, ტექსტურები, მარტივი გეომეტრიული ფორმები — უნივერსალური აღმოჩნდეს და წარმატებით შეძლოს ადამიანების სახეებზე ემოციების ამოცნობაც. 

**რეალური შედეგი:**
```
Train: 55.59%
Val : 57.15%
Test : 58.21%
```

**ანალიზი:**
გაყინულმა ResNet18-მა აჩვენა ბევრად უარესი შედეგი, ვიდრე ნულიდან აწყობილმა მე-3 ექსპერიმენტმაც კი (SmallCNN: 64.17%).
- Domain Gap: ImageNet-ის მონაცემთა ბაზა შედგება მაღალი რეზოლუციის, ფერადი სურათებისა და მრავალფეროვანი ობიექტებისგან(ცხოველები, მანქანები და ა.შ.). FER2013-ის მონაცემები კი სპეციფიკური, დაბალი რეზოლუციის (48x48), შავ-თეთრი სახეებია. მონაცემებს ეს სხვაობა იმდენად დიდია, რომ წინასწარ ნასწავლი მახასიათებლები ახალი ამოცანისთვის არარელევანტური აღმოჩნდა.
- ტრენინგის დაბალი სიზუსტე (55.59%) მიუთითებს მძლავრ Underfitting-ზე. კლასიფიკატორის 500K პარამეტრი არ აღმოჩნდა საკმარისი.

გადაწყვეტილება:
backbone freezing ამ ამოცანისთვის არაეფექტურია. შემდეგი აუცილებელი ნაბიჯია მოდელის სრული Fine-Tuning, რათა მთელმა არქიტექტურამ შეძლოს წონების ადაპტირება უშუალოდ სახის ემოციების მახასიათებლებზე.

### ექსპერიმენტი 9 — ResNet18 სრული Fine-Tune

- სტრუქტურა: იდენტური ექსპერიმენტი 8-ის, თუმცა ამჯერად გაუქმებულია გაყინვა (freeze_backbone=False).
- პარამეტრების რაოდენობა: ~11,400,000 (მთლიანი ქსელი იწვრთნება).

კონფიგურაცია:
- lr=0.0001: მე-8 ექსპერიმენტთან შედარებით 10-ჯერ შემცირებული სწავლების ტემპი. ეს კრიტიკულია კატასტროფული დავიწყების (Catastrophic Forgetting) თავიდან ასაცილებლად — მაღალი LR უბრალოდ წაშლიდა ImageNet-ზე ნასწავლ სასარგებლო წონებს. მცირე LR უზრუნველყოფს მათ ფრთხილ და ნელ ადაპტაციას.
- optimizer=AdamW, weight_decay=0.01: ვინაიდან მოდელი მძლავრია (11.4 მილიონი პარამეტრი 28,000 სურათისთვის), Overfitting-ის რისკი ექსტრემალურად მაღალია. ამიტომ გამოყენებულია გაძლიერებული L2 რეგულარიზაცია.
- ეპოქები=50: სრული Fine-Tuning დაბალი სწავლების ტემპით მოითხოვს მეტ იტერაციას წონების ოპტიმალური განახლებისთვის.
- scheduler=CosineAnnealing: უზრუნველყოფს ტრენინგის ბოლო ეტაპზე LR-ის გლუვ შემცირებას და მოდელის დასტაბილურებას.

ჰიპოთეზა:
სავარაუდოდ, დაბალი LR-ით მთელი ქსელის ტრენინგი საშუალებას მისცემს მოდელს, ImageNet-ის ბაზაზე ნასწავლი ზოგადი მახასიათებლები იდეალურად მოარგოს ადამიანის სახის ემოციების სპეციფიკას. შეიძლება ამ მოდელმა გადააჭარბოს DeepCNN-ის (Exp 7) შედეგებს და დააფიქსიროს საუკეთესო მაჩვენებელი.

**რეალური შედეგი:**
```
Train : 84.04%
Val : 64.06%
Test : 63.81%
```

**ანალიზი:** 
- მიუხედავად იმისა, რომ სრულმა Fine-Tuning-მა გაყინულ ვერსიას (Exp 8) +7%-ით აჯობა, ის მაინც მნიშვნელოვნად ჩამორჩა  ნულიდან აწყობილ DeepCNN-ს (63.81% vs 69.63%). ეს მიუთითებს იმაზე, რომ Transfer Learning ყოველთვის არ არის საუკეთესო გამოსავალი.

მოდელის წარუმატებლობის მთავარი მიზეზები: 
- conv1-ის ცვლილება: იმისათვის, რომ მოდელს შავ-თეთრი სურათები მიეღო, ჩვენ თავდაპირველი 3-არხიანი (RGB) ფენა შემთხვევითი წონებით (Random Initialization) ჩავანაცვლეთ. ამან გამოიწვია ის, რომ პირველ ფენაში სრულად განადგურდა საბაზისო მახასიათებლების (კიდეების, ფერების) დეტექტორები. მომდევნო პრეტრენინგებულ ფენებს აღარ მიეწოდებოდათ ინფორმაცია იმ ფორმატში, როგორშიც ელოდნენ.
- სივრცული რეზოლუციის შეუთავსებლობა: ResNet18 გაწვრთნილია 224×224 ზომის სურათებისთვის. ჩვენი მონაცემები კი 48×48 ზომისაა. ResNet-ის Pooling-ის ფენები ამ მცირე სურათის სივრცულ განზომილებებს ზედმეტად სწრაფად ამცირებს. შედეგად, ქსელის ბოლო ფენებში სურათი ფაქტობრივად წერტილამდე დადის (1x1), რაც კარგავს ემოციების გამოსაცნობად საჭირო ინფორმაციას (მაგ. პირის და თვალების პოზიციებს).
- ტრენინგის ხანგრძლივობა: conv1-ის ნულიდან სწავლებას და დანარჩენი ქსელის ადაპტირებას LR=0.0001 პირობებში შესაძლოა 50 ეპოქაზე გაცილებით მეტი დრო დასჭირდეს.

გადაწყვეტილება:
ImageNet-ზე პრეტრენინგებული მოდელები მცირე ზომის (48x48) და ერთარხიანი სურათებისთვის არაეფექტურია, რადგან არქიტექტურის მოდიფიკაცია Transfer Learning-ის უპირატესობებს აქრობს. ექსპერიმენტების სერიამ აჩვენა, რომ FER2013 ამოცანისთვის DeepCNN (ჩვენი Custom მოდელი Residual კავშირებითა და GAP-ით) წარმოადგენს ყველაზე ოპტიმალურ, ძლიერ და სტაბილურ მოდელს.

| 7 | **exp7_deepcnn_residual** | DeepCNN | AdamW | 1e-3 | 70 | 32 | **68.99%** | **69.63%** | **საუკეთესო** |
| 8 | exp8_resnet18_frozen | ResNet18 frozen | Adam | 1e-3 | 20 | 32 | 57.15% | 58.21% | Transfer შეზღუდული |
| 9 | exp9_resnet18_finetune | ResNet18 full | AdamW | 1e-4 | 50 | 32 | 64.06% | 63.81% | მოულოდნელად ნაკლები |

## შეჯამება

### 1. Underfitting vs Overfitting (Exp 1 vs 2)
TinyNet (50.63%) — train და val ორივე დაბალი, მცირე სხვაობა = underfitting. SmallCNN regularization-ის გარეშე — train ~80% vs val 60.24% = overfitting. ეს ორი ექსპერიმენტი bias-variance tradeoff-ს ნათლად აჩვენებს.

### 2. Regularization ეფექტურია (Exp 2 vs 3)
იდენტურ არქიტექტურაზე Dropout + augmentation-ის დამატებამ **+4% val სიზუსტე** მოიტანა. Augmentation განსაკუთრებით ღირებულია FER2013-ის შეზღუდული dataset-ისთვის.

### 3. სიღრმე ეხმარება (Exp 3 → 4 → 7)
Val სიზუსტის პროგრესი: 64.17% → 66.51% → 68.99%. სიღრმის ყოველი ზრდა (სწორი regularization-ით) სიზუსტეს სტაბილურად აუმჯობესებდა.

### 4. Learning Rate კრიტიკულია (Exp 4 vs 6)
LR-ის ცვლილებამ 0.001-დან 0.01-მდე **14%-იანი ვარდნა** გამოიწვია (66.51% → 52.66%). ეს ყველა ექსპერიმენტს შორის ყველაზე დრამატული ერთ-ცვლადიანი პოვნაა. Adam-ისთვის lr ∈ [1e-4, 3e-3] რეკომენდებული range-ია.

### 5. Adam SGD-ს სჯობს (Exp 4 vs 5)
ამ ამოცანაში Adam SGD+Nesterov-ს 3.12%-ით სჯობს (66.51% vs 63.39%). მცირე dataset-ებზე Adam-ის ადაპტური learning rate-ები მნიშვნელოვან უპირატესობას იძლევა.

### 6. Residual კავშირები გადამწყვეტია (Exp 4 vs 7)
Skip კავშირებმა MediumCNN-ზე +2.5% გაუმჯობესება გამოიწვია. GAP overfitting-ს ამცირებს, AdamW + ძლიერი weight decay კი კარგი კომბინაციაა ღრმა ქსელებისთვის.

### 7. Transfer Learning ამ ამოცანაზე შეზღუდულია (Exp 7 vs 9)
Custom DeepCNN (68.99%) pretrained ResNet18-ს (64.06%) სჯობს. ძირეული მიზეზები: conv1-ის შეცვლამ pretrained ნიშნები გაანადგურა; 48×48 input ResNet-ის 224×224-ზე გათვლილ არქიტექტურასთან შეუთავსებელია. Exp 10 ამ პრობლემებს აგვარებს.
